🗨️ 웹소켓 채팅

📝 프로젝트 개요

Websocket을 이용해 구현한 채팅 어플리케이션입니다. 닉네임 설정, 알람끄기 기능 등을 구현했으며, 실시간 채팅이 가능하도록 구현했습니다.

🛠️ 사용 기술

  • Websocket
  • NodeJS

📌 주요 기능

  • 라이브 채팅
  • 새 메시지 도착 시document.title에 알람
  • 이전 메시지 UX 개선

🔍 개발 포인트

  • Websocket

    Websocket 기능을 활용해 라이브 채팅이 가능한 실시간 통신 기능을 구현하였습니다.

  • 유저메시지, 서버메시지 구분

    WebSocket은 기본적으로 텍스트 기반 통신만 지원하기 때문에, 알림 메시지와 유저 채팅 메시지를 구분하기 위해 JSON 포맷을 사용해 데이터를 전송하고, 클라이언트에서 파싱하여 처리했습니다.

  • UX를 고려한 개발

    초기 개발에서 새 메시지 도착 시 채팅창이 항상 아래로 이동하도록 동작하여, 만약 읽지 않은 메시지가 쌓여서 이전 메시지를 확인하는 도중 새 메시지가 도착한다면 스크롤이 강제로 최하단으로 내려가버리는 현상이 발생했습니다. 사용자가 보고있는 창의 상태를 체크하고, 이전 메시지를 확인하는 중인 경우 스크롤이 하단으로 이동하지 않도록 client 측에서 처리하였습니다.

💭 느낀 점

WebSocket을 사용하여 실시간 채팅 서비스를 구현하여 서비스하며 http 양방향 통신에 대한 이해도를 높일 수 있었습니다. 또한 WebSocket의 단순한 텍스트 통신 구조 속에서 JSON을 활용한 메시지 타입 구분 로직을 설계하며, 데이터 인코딩 및 파싱에 대한 이해도를 높일 수 있는 좋은 경험이었습니다. 개발 단계에서는 미처 생각하지 못했던 UX적인 부분에서 여러 사람들과 함께 테스트하며 부족한 부분을 발견하면서 협업과 피드백의 중요성을 배울 수 있었습니다.
